## 推送前准备

### 1. 在 GitHub 网页上手工新建仓库

打开 https://github.com/new ，填写：

| 字段 | 值 |
| --- | --- |
| Owner | `billysince` |
| Repository name | `GongwenAssistant` |
| Description | `WPS 公文助手 (公文助手 v2.4.1 的源码级开源重制版)` |
| Visibility | Public（推荐，开源透明） |
| Initialize this repository with | **全部不勾选**（README/LICENSE/.gitignore 全留空，我们本地已经有） |

点 `Create repository`。**别让 GitHub 自动加任何文件**，否则本地第一次 push 会冲突。

### 2. 准备 GitHub 凭证（二选一）

#### 方案 A：Personal Access Token (PAT)

1. 到 https://github.com/settings/tokens 生成 classic token，至少勾 `repo` 范围。
2. 复制 token，下次 `git push` 时 username 填 `billysince`、password 粘 token。
3. 如果想免去每次输入，开启 Windows Credential Manager 缓存：
   ```powershell
   git config --global credential.helper manager-core
   ```

#### 方案 B：SSH（更稳定）

```powershell
ssh-keygen -t ed25519 -C "yongchen@mail2.gdut.edu.cn"
type $env:USERPROFILE\.ssh\id_ed25519.pub
```

把输出贴到 https://github.com/settings/keys 新增 SSH key。
然后把 remote 切换到 SSH：

```powershell
cd 'D:\工作\20260520\GongwenAssistant'
& 'C:\Program Files\Git\cmd\git.exe' remote set-url origin git@github.com:billysince/GongwenAssistant.git
```

## 如果出错

| 错误 | 原因 | 修法 |
| --- | --- | --- |
| `fatal: refusing to merge unrelated histories` | GitHub 自动建了 README/LICENSE 占了空仓库 | 删远端仓库重建（**不要**勾任何初始化选项） |
| `error: failed to push some refs ... fetch first` | 同上 | 同上 |
| `Authentication failed` | PAT 过期或没勾 repo 权限 | 重新到 settings/tokens 生成 |
| `RPC failed, ... 408` | 网络慢，50 MB 一次推不完 | 用 `--no-thin`，或先 push 部分历史 |
| 中文文件名在 GitHub 显示 `公文助手` 而不是 `公文助手` | git 默认 quote 非 ASCII | 已在 .git/config 设 `core.quotepath = false`，应该 OK |
