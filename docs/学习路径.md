## 路径一：5 分钟极速版（"我就想知道这是干嘛的"）

读这一份就够：

| 顺序 | 文件 | 重点 |
|---|---|---|
| 1 | [README.md](../README.md) §一 + §二 | 一句话 + 状态卡片 |
| 2 | [README.md](../README.md) §三 | 真实截图 |

- 这是个 WPS 顶部菜单插件

## 不同背景的人, 阅读侧重点

### 文秘 / 公务员 / 审计岗（无开发背景）

- 看 路径一 + [QUICKSTART.md](QUICKSTART.md) 装一下试试
- 关心的是 F1 / F2 公文 docx 出不出, 不需要研究 IL
- 推荐直接用 `审计工作20260521/F-公文版/*.docx` 成品, 或用 markdown + Pandoc 自己转

### 桌面运维 / IT 支持

- 走 路径二
- 重点关注 [INSTALL.md](INSTALL.md) 和 [SECURITY.md](SECURITY.md)
- 准备好回答用户问题：怎么装 / 怎么卸 / 杀软怎么办 / 出错怎么办

### .NET 开发者（COM Add-in / VSTO 经验）

- 跳过术语表, 直接走 路径三
- 关注点：Harmony 怎么用、为什么不修改原 dll、HKCU + CodeBase 注册细节
- 看完后可以参考 [踩坑全集.md](踩坑全集.md) 学习反面案例

### 逆向工程 / 安全研究

- 直接走 路径四
- 关注点：dnlib 改 IL vs Harmony 在内存 hook 的取舍、PE Import Table 一致性、SHA256 字节比对、杀软启发式触发模式
- 看 [踩坑全集.md](踩坑全集.md) #23 #24 #25 三条核心限制

### WPS 内核 / 国产 Office 研究者

- 重点看 [踩坑全集.md](踩坑全集.md) #25 #26
- 这两条记录了 WPS 12.1+ 个人版对 COM 和 JS 双加载机制的安全限制
- 我们没找到绕过办法, 你如果找到了非常欢迎在 Issue 区分享
